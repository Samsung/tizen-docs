Introduction

The Eio library is a library that implements an API for asynchronous input/output operation. Most operation are done in a separated thread to prevent lock. See Eio_Group. Some helper to work on data received in Eio callback are also provided see Eio_Helper. It is also possible to work asynchronously on Eina_File with Eio_Map or on Eet_File with Eio_Eet. It come with way to manipulate eXtended attribute assynchronously with Eio_Xattr.

This library is cross-platform and can be compiled and used on Linux, BSD, Opensolaris and Windows (XP and CE). It is heavily based on Ecore main loop.

How to compile

Eio is a library your application links to. The procedure for this is very simple. You simply have to compile your application with the appropriate compiler flags that the pkg-config script outputs. For example:

Compiling C or C++ files into object files:

   gcc -c -o main.o main.c `pkg-config --cflags eio`

Linking object files into a binary executable:

   gcc -o my_application main.o `pkg-config --libs eio`

See pkgconfig

Next Steps

After you understood what Eio is and installed it in your system you should proceed understanding the programming interface.

Recommended reading:

  • Eio_Helper for common functions and library initialization.
  • Eio_Map to manipulate files asynchronously (mmap).
  • Eio_Xattr to access file extended attributes (xattr).
  • Eio_Monitor to monitor for file changes (inotify).
  • Eio_Eet to access Eet files asynchronously.
