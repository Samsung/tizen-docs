Media Controller Server
Media Controller

The Media Controller Server API provides functions for sending the server information to the client.

Required Header

#include <media_controller_server.h>

Overview

The Media Controller Server API allows you to send the playback info (mc_server_set_playback_state(), mc_server_set_playback_position(), mc_server_set_playlist_item_index()), or the metadata (mc_server_set_metadata()). and to receive the custom command (mc_server_set_custom_command_received_cb()), to update the status information on the latest server (mc_server_update_playback_info(), mc_server_update_metadata()), or to create/destroy the handle (mc_server_create()/mc_server_destroy()).

typedef void(* mc_server_custom_command_received_cb)(const char *client_name, const char *command, bundle *data, void *user_data)

Called when the Server receives custom command from the client.

This callback is called when server receives custom command from client. If there is reply for command, call mc_server_send_command_reply() function.

Since :
3.0
Remarks:
The callback is called in the dbus event loop. data should be released using bundle_free().
Parameters:
[in]client_nameThe app_id of the media controller client. It can be used only in the callback. To use outside, make a copy.
[in]commandThe received command. It can be used only in the callback. To use outside, make a copy.
[in]dataThe extra data
[in]user_dataThe user data passed from the mc_server_set_custom_command_received_cb() function
Precondition:
mc_server_set_custom_command_received_cb()
See also:
mc_server_set_custom_command_received_cb()

int mc_server_update_playlist ( mc_server_h  server,
mc_playlist_update_mode_e  mode,
const char *  playlist_name,
bundle *  data 
)

Updates the modified playlist info.

If this function is called, the updated playlist will be sent to all clients.
The server can register only one playlist. If a new one is registered, the old one is removed.
data is a bundle. A bundle is a set of key-value pairs. "index" is the key and "media metadata" is the value.
Therefore you should fill the metadata order using mc_meta_e.
If there are no playlist items or the server wants to remove the playlist, set data to NULL.

5.0
